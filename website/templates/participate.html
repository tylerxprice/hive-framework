Participate

The Hive Bot Tournament is currently open to everyone. Available computing power may limit the number of entrants in the future, but if you are interested in building a bot read on.

Getting Started

If you are not already familiar with Hive you can read the rules on the official site [PDF] or jump right in and play online BoardSpace.net.

All the code that runs the hive games including scheduling matchups, validing moves, and checking for winners is open-source and available on the github repository. The Python script "framework.py" can be used to play a game of hive in three combinations: human versus human, human versus bot, or bot versus bot. Also a sample bot "randy.py" is also provided in the repo. This bot only searches one move ahead and gives a random score to the board position.

Note: the framework is still in development, however the core functionality is in place.

Building Your Bot

Even though the framework is programmed in Python, this should in no way limit what language you use. The framework will invoke your bot as a subprocess passing the game state through the command line and reading the move from the output. Currently bots are being worked on in Python, Ruby, and C.

The specification for the bots is maintained on the github wiki page.

Submitting Your Bot

Currently there is no way to submit your bot and have it automatically show up in the tournament. You can e-mail me your source code and build files in an compressed archive along with any notes for specific versions or extensions needed to build it. I will then manually add the bot to the tournament.
